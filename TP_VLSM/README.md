# TP VLSM – Architecture Cisco Packet Tracer

Ce dossier contient le travail pratique réalisé dans le cadre du module **Réseaux Informatiques**, portant sur l’**adressage IP avec VLSM (Variable Length Subnet Mask)** et la **simulation d’une architecture réseau dans Cisco Packet Tracer**.  

---

## Objectifs pédagogiques

- Comprendre et appliquer le concept de **VLSM**  
- Calculer les sous-réseaux adaptés à différents besoins en nombre d’hôtes  
- Déterminer les adresses réseau, broadcast et passerelle pour chaque sous-réseau  
- Configurer une architecture réseau dans Cisco Packet Tracer avec routeurs, switches et PCs  
- Vérifier la connectivité entre les sous-réseaux avec des commandes `ping`

---

## Contenu du TP

### 1️⃣ Calcul VLSM

**Description :**  
- Division d’un réseau principal en plusieurs sous-réseaux de tailles différentes selon les besoins en hôtes  
- Calcul précis des éléments suivants pour chaque sous-réseau :  
  - Adresse réseau  
  - Masque de sous-réseau (subnet mask)  
  - Plage d’adresses utilisables  
  - Adresse de broadcast  
  - Adresse de la passerelle (gateway)

**Objectifs spécifiques :**  
- Optimiser l’utilisation des adresses IP  
- Préparer la configuration des équipements dans Cisco Packet Tracer  
- Documenter les sous-réseaux dans un tableau récapitulatif

**Fichiers associés :**  
- `TPVLSMBH.pdf` : rapport avec calculs et explications  

---

### 2️⃣ Architecture réseau dans Cisco Packet Tracer

**Description :**  
- Mise en place de l’architecture réseau en utilisant Cisco Packet Tracer  
- Configuration des routeurs, switches et PCs  
- Attribution des adresses IP selon les sous-réseaux calculés  
- Test de connectivité entre sous-réseaux via ping

**Objectifs spécifiques :**  
- Configurer les interfaces des routeurs et PCs correctement  
- Vérifier la communication entre tous les sous-réseaux  
- Détecter et corriger les erreurs de configuration éventuelles

**Fichiers associés :**  
- `VLTPVLSMBH.pkt` : fichier Cisco Packet Tracer
